# Security & Non-Custodial Design

PolyNotice is built with a non-custodial architecture.

## Principles

- No private keys or seed phrases are collected
- Funds remain in the user's wallet
- Automation is opt-in and permission-based
- Permissions are limited and revocable

## Automation Model (Planned)

For automation features:
- Users approve specific actions
- Execution is limited by defined conditions
- Trades settle directly on the underlying protocol

PolyNotice does not custody user funds at any point.
